Use these as starting points for periodic persona refreshes. Re-run searches every cycle because skill/MCP recommendations change quickly.

## Roles To Check

- Frontend: UI generation, React/Next.js, component scanning, browser testing, accessibility.
- Backend: API contracts, database tools, security review, current library docs, debugging.
- Fullstack: cross-layer feature workflow, browser testing, DB inspection, docs lookup, security.
- PM: issue/PR triage, release notes, PRDs, status updates, codebase summarization.
- QA: Playwright/browser automation, test planning, data generation, accessibility, API testing.
- Designer: Figma handoff, UI polish, design systems, accessibility, visual QA.
- Technical Writing: docs generation, docs management, DOCX/PDF, release notes, code exploration.

## Source Quality Notes

- Prefer exact skill or MCP pages when available.
- Use directory/search/category pages only when exact pages are unstable or unavailable.
- Use community/blog recommendations as supporting evidence, not the only basis for a bundle.
- Do not cite pages that cannot be opened or verified during the refresh.
